Environment Setup

This guide will help you set up your development environment for the course. All code examples use Python with NumPy, scikit-learn, TensorFlow/Keras, and Matplotlib.

1. Python Installation

We recommend Python 3.9 or later. You can download Python from:

  • python.org (official)
  • Anaconda (includes many scientific packages)

Verify your installation:

python --version

2. Virtual Environment (Recommended)

Create an isolated environment for this course:

# Create virtual environment
python -m venv datasci207

# Activate it (macOS/Linux)
source datasci207/bin/activate

# Activate it (Windows)
datasci207\Scripts\activate

3. Required Libraries

Install the core libraries:

pip install numpy scikit-learn tensorflow matplotlib
Library Purpose Used In
numpy Numerical computing, arrays All modules
scikit-learn Classical ML algorithms Modules 1-8, 12
tensorflow Deep learning (includes Keras) Modules 6, 9-11, 13
matplotlib Basic plotting Various modules

4. Running Code Files

Each module includes .py files with heavily commented code. To run them:

# Navigate to the module directory
cd week_01

# Run the lecture code
python lecture.py

The code files are designed to be read top-to-bottom. Comments explain each concept as it appears. You can also copy sections into an interactive Python session to experiment.

5. GPU Setup (Optional)

For the deep learning modules (6, 9-11, 13), training will be faster with GPU support. This is optional; all examples will run on CPU.

If you have an NVIDIA GPU:

# Install GPU-enabled TensorFlow
pip install tensorflow[and-cuda]

Verify GPU is detected:

python -c "import tensorflow as tf; print(tf.config.list_physical_devices('GPU'))"

Troubleshooting

Common issues and solutions:

ImportError: No module named 'X'

Make sure your virtual environment is activated and run pip install X.

TensorFlow installation fails

Try: pip install --upgrade pip then reinstall TensorFlow.

Permission denied errors

On macOS/Linux, you may need to use pip install --user.
